# 本地编辑器要求

本文档由 `xiaoge-static-blog-plan.md` 拆分整理，用于约束小鸽志本地可视化编辑器的开发范围。编辑器是本地工具，只负责读写静态博客项目中的 Markdown、YAML、图片和 Git 操作，不作为公网后台部署。

## 1. 项目目标

开发一个只在本机运行的可视化编辑器，用于管理小鸽志静态博客内容，降低手写 Markdown、维护 frontmatter、上传图片和执行 Git 命令的操作成本。

核心目标：

- 使用 Python + FastAPI 实现。
- 只监听 `127.0.0.1`。
- 不使用数据库。
- 直接读写本地 Markdown、YAML 和图片文件。
- 支持文章新建、编辑、删除、预览。
- 支持分类、标签配置维护。
- 支持图片上传和压缩。
- 支持本地构建。
- 支持 Git commit 和 Git push。
- 不部署到公网。

## 2. 技术选型

必须使用：

```text
Python + FastAPI
```

推荐原因：

- 适合开发本地工具。
- 读写 Markdown 文件方便。
- 读写 YAML 配置方便。
- 调用图片处理、构建命令和 Git 命令方便。
- 无需数据库。
- 可以通过浏览器提供可视化编辑界面。

## 3. 运行要求

默认访问地址：

```text
http://127.0.0.1:18787
```

监听要求：

```text
只允许监听 127.0.0.1
不得监听 0.0.0.0
不得默认暴露到局域网或公网
```

编辑器目录建议：

```text
tools/editor/
├── main.py
├── requirements.txt
├── templates/
└── static/
```

## 4. 数据边界

编辑器不得引入数据库。所有数据来自博客项目文件：

```text
site/src/content/posts/
site/src/content/data/categories.yml
site/src/content/data/tags.yml
site/src/content/data/menu.yml
site/src/content/data/links.yml
site/public/
```

编辑器只负责文件读写，不改变博客主体的数据规范。

## 5. 文章管理要求

### 5.1 文章列表

文章列表接口需要读取：

```text
site/src/content/posts/*/index.md
```

列表项至少包含：

- 标题。
- slug。
- 发布时间。
- 主分类。
- 标签。
- 封面图。
- SEO 描述。
- 是否置顶。
- 是否启用评论字段。

列表功能要求：

- 支持按发布时间倒序。
- 支持按标题搜索。
- 支持按分类筛选。
- 支持按标签筛选。
- 支持显示缺失字段提醒。

### 5.2 新建文章

新建文章时必须创建：

```text
site/src/content/posts/{slug}/index.md
site/src/content/posts/{slug}/img/
```

新建表单至少包含：

- 标题。
- slug。
- 发布时间。
- 分类。
- 标签。
- SEO 描述。
- 封面图。
- 置顶权重。
- Markdown 正文。

slug 生成要求：

- 支持根据标题自动生成英文 slug。
- 支持手动修改 slug。
- 保存前必须检查 slug 是否重复。
- slug 只能用于生成 `/archives/{slug}`，不得生成其他文章 URL 结构。

### 5.3 编辑文章

编辑文章时必须保持 Markdown frontmatter 结构稳定。

可编辑字段：

```text
title
slug
description
date
categories
tags
cover
top
comments
正文 Markdown
```

编辑要求：

- 修改 slug 时必须同步检查目录名和 URL 冲突。
- 修改分类时只能选择已有分类，或提示先维护分类配置。
- 修改标签时可选择已有标签，也可按规则新增标签配置。
- 保存前必须校验必填字段。
- 保存后文件仍为可读 Markdown。

### 5.4 删除文章

删除文章是高风险操作，必须满足：

- 删除前展示文章标题、slug、目录路径。
- 要求二次确认。
- 默认不物理删除，可先移动到本地回收目录或归档目录。
- 不得误删 `site/src/content/posts/` 之外的文件。

## 6. Frontmatter 处理要求

编辑器必须支持以下字段：

```yaml
title: ""
slug: ""
description: ""
date: ""
categories: []
tags: []
cover: ""
top:
comments: true
```

字段校验：

| 字段 | 校验规则 |
| --- | --- |
| `title` | 必填，不能为空 |
| `slug` | 必填，只能包含小写字母、数字和连字符 |
| `description` | 建议填写，用于 SEO |
| `date` | 必填，保存时统一为 `YYYY-MM-DD` |
| `categories` | 至少 1 个 |
| `tags` | 可为空 |
| `cover` | 可为空；填写时必须指向存在的图片 |
| `top` | 可为空；填写时必须为数字 |
| `comments` | 布尔值；首期仅保存字段，不实现评论功能 |

## 7. Markdown 编辑要求

编辑界面必须提供：

- Markdown 原文编辑。
- Markdown 预览。
- 标题、列表、引用、代码块、表格、图片等基础渲染。
- 保存前预览 frontmatter 和正文组合后的效果。

图片插入要求：

- 上传正文图片后自动插入相对路径。
- 相对路径格式优先为 `./img/{filename}`。
- 不应自动把新文章图片放入公共上传目录。

## 8. 图片管理要求

### 8.1 封面上传

封面上传目标目录：

```text
site/src/content/posts/{slug}/img/
```

默认文件名：

```text
cover.webp
```

要求：

- 支持 jpg、jpeg、png、webp 输入。
- 上传后优先转换或压缩为 webp。
- 保存后更新 frontmatter.cover 为 `./img/cover.webp`。
- 如果已存在封面，覆盖前需要确认。

### 8.2 正文图片上传

正文图片上传目标目录：

```text
site/src/content/posts/{slug}/img/
```

要求：

- 支持自定义文件名。
- 支持自动清理文件名中的空格和特殊字符。
- 支持图片压缩。
- 上传后返回 Markdown 图片引用文本。

示例：

```md
![线路测试](./img/route-test.webp)
```

### 8.3 旧图片限制

编辑器首期不负责批量迁移旧图片路径。旧图片路径整理属于博客迁移任务，不作为本地编辑器首期功能。

## 9. 分类管理要求

分类配置文件：

```text
site/src/content/data/categories.yml
```

编辑器必须支持：

- 查看分类列表。
- 新增分类。
- 编辑分类名称和描述。
- 删除未被文章使用的分类。
- 统计每个分类下的文章数量。

分类字段：

```yaml
- slug: vps-discount
  name: VPS优惠
  description: VPS 优惠活动、促销套餐、限时折扣
```

删除限制：

- 如果分类已被文章引用，不允许直接删除。
- 必须先修改相关文章分类后再删除。

## 10. 标签管理要求

标签配置文件：

```text
site/src/content/data/tags.yml
```

编辑器必须支持：

- 查看标签列表。
- 新增标签。
- 编辑标签显示名称。
- 删除未被文章使用的标签。
- 统计每个标签下的文章数量。

标签字段：

```yaml
- slug: cloudflare
  name: Cloudflare
```

删除限制：

- 如果标签已被文章引用，不允许直接删除。
- 必须先从相关文章中移除标签后再删除。

## 11. 推荐 API

### 11.1 文章 API

```text
GET    /api/posts
GET    /api/posts/{slug}
POST   /api/posts
PUT    /api/posts/{slug}
DELETE /api/posts/{slug}
```

### 11.2 分类 API

```text
GET /api/categories
PUT /api/categories
```

### 11.3 标签 API

```text
GET /api/tags
PUT /api/tags
```

### 11.4 图片 API

```text
POST /api/upload-image
```

上传参数建议：

```text
postSlug
type: cover | content
file
alt
filename
```

### 11.5 构建和 Git API

```text
POST /api/build
POST /api/git-commit
POST /api/git-push
```

## 12. 本地构建要求

编辑器需要提供“本地构建”操作。

执行位置：

```text
site/
```

执行命令：

```text
npm run build
```

要求：

- 展示构建中状态。
- 展示构建成功或失败结果。
- 构建失败时展示关键错误日志。
- 不自动吞掉错误。

## 13. Git 操作要求

编辑器可提供一键 Git 操作，但必须谨慎。

### 13.1 Git 状态

提交前必须展示：

- 已修改文件。
- 新增文件。
- 删除文件。

### 13.2 Git commit

提交要求：

- 用户填写 commit message。
- 不允许空 message。
- 提交前展示将要提交的文件列表。
- 不自动提交敏感文件。

### 13.3 Git push

推送要求：

- push 前确认当前分支。
- push 失败时显示错误原因。
- 不在编辑器中明文保存 GitHub Token。

## 14. 安全要求

编辑器必须满足：

- 只监听 `127.0.0.1`。
- 不提供公网部署说明作为默认方案。
- 不写入项目目录之外的文件，除非用户显式配置。
- 不保存 GitHub Token、OpenAI API Key、登录密码等敏感信息。
- 不上传 `.env`、日志、缓存、临时文件。
- 删除、移动、覆盖文件前必须校验路径位于项目允许目录内。

建议 `.gitignore`：

```gitignore
# 本地编辑器
tools/editor/

# 本地环境变量
.env
.env.local

# 日志
*.log

# 临时文件
.tmp/
.cache/
```

## 15. 非目标

本地编辑器首期不包含：

- 公网后台。
- 用户登录系统。
- 多用户协作编辑。
- 数据库。
- 评论管理。
- 阅读数管理。
- 点赞管理。
- 批量旧图迁移。
- 在线部署平台管理。

## 16. 验收标准

开发完成后至少满足：

- 访问 `http://127.0.0.1:18787` 可以打开编辑器。
- 服务没有监听 `0.0.0.0`。
- 可以列出已有 Markdown 文章。
- 可以新建文章并生成正确目录结构。
- 可以编辑并保存 frontmatter 和正文。
- 可以上传封面并写入 `cover` 字段。
- 可以上传正文图片并插入相对路径。
- 可以维护 `categories.yml`。
- 可以维护 `tags.yml`。
- 可以执行本地构建并显示结果。
- 可以查看 Git 状态、提交和推送。
- 不会写入或删除项目目录之外的文件。
