# 页面结构需求

本文件用于整理用户自定义页面内容。正式页面写在 `blog/pages/`；`example/pages/` 只放模板和空内容时的兜底示例，用户不要直接在 `example/` 里维护正式内容。

## 目录结构

每个自定义页面一个独立目录，目录名就是页面 URL slug。当前只支持一级页面路径。

```text
blog/pages/
└─ {slug}/
   ├─ index.md
   └─ img/              # 可选，存放当前页面正文图片
      ├─ {slug}-1.webp
      └─ {slug}-2.webp
```

示例：

```text
blog/pages/
└─ about/
   ├─ index.md          # 生成 /about
   └─ img/
      └─ about-1.webp
```

页面目录名建议只使用小写字母、数字和短横线，例如 `about`、`projects`、`contact-me`。

## 文件命名

| 文件 | 要求 |
| --- | --- |
| `blog/pages/{slug}/index.md` | 页面正文和属性区。 |
| `blog/pages/{slug}/img/` | 可选页面图片目录。 |
| `img/{slug}-1.webp` | 建议的正文第 1 张图片命名。 |
| `img/{slug}-2.webp` | 建议的正文第 2 张图片命名，后续按顺序递增。 |

## 属性区

Markdown 文件开头两条 `---` 之间是属性区。页面属性区只需要站内页面本身的信息，不写文章字段。

最小完整示例：

```md
---
title: "关于"
description: "关于这个站点和内容方向。"
date: "2026-06-09"
comments: true
---

这里写页面正文。页面正文同样支持 Markdown。

## 页面小标题

页面和文章一样，会根据 `##`、`###`、`####` 自动生成右侧目录，并支持滚动高亮。

![页面图片说明](./img/about-1.webp)
```

| 字段 | 要求 |
| --- | --- |
| `title` | 页面标题，必填。 |
| `description` | 页面摘要，用于 SEO。 |
| `date` | 页面日期，使用 `YYYY-MM-DD`。 |
| `comments` | 是否开启评论，默认 `true`。即使省略该字段，也按开启处理；只有显式写 `false` 时才关闭。当前只保留字段，不开发评论 UI。 |

页面不要写 `slug`、`categories`、`tags`、`cover`、`top`。这些字段只属于文章。

## 页面目录

页面正文和文章正文使用同一套目录功能：

1. `##`、`###`、`####` 会进入右侧目录。
2. 没有这些标题时，目录区域不会显示条目。
3. 目录高亮会跟随页面滚动更新。

## 正文图片

页面专用图片放在当前页面目录的 `img/` 中，用相对路径引用：

```md
![图片说明](./img/about-1.webp)

![图片说明](./img/about-2.webp)
```

如果图片会被多个页面或文章复用，可以放在 `blog/public/`。该目录构建后使用根路径访问，例如 `blog/public/assets/banner.webp` 在 Markdown 中写作 `/assets/banner.webp`。

## 示例页面

`example/pages/demo-page/` 是兜底示例页面。只有当 `blog/pages/` 没有任何 `index.md` 时，才会构建示例页面 `/demo-page`。一旦用户添加自己的页面，示例页面就不再生成。
