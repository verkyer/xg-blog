# 页面结构需求

本文件用于迁移自定义页面内容。只描述页面目录、Markdown frontmatter 和图片放置规则。

## 目录结构

每个自定义页面一个独立目录，目录名就是页面 URL slug。当前只支持一级页面路径。

```text
blog/pages/
└─ {slug}/
   ├─ index.md
   └─ img/              # 可选，存放当前页面正文图片
      ├─ {slug}-1.webp
      └─ {slug}-2.webp
```

示例：

```text
blog/pages/
└─ about/
   ├─ index.md          # 生成 /about
   └─ img/
      └─ content.png
```

## 文件命名

| 文件 | 要求 |
| --- | --- |
| `blog/pages/{slug}/index.md` | 页面正文和 frontmatter。 |
| `blog/pages/{slug}/img/` | 可选页面图片目录。 |
| `img/{slug}-1.webp` | 建议的正文第 1 张图片命名。 |
| `img/{slug}-2.webp` | 建议的正文第 2 张图片命名，后续按顺序递增。 |

页面目录名建议只使用小写字母、数字和短横线，例如 `about`、`projects`、`contact-me`。

## Frontmatter

页面 frontmatter 只需要站内页面本身的信息，不写文章字段。

```md
---
title: "关于"
description: "关于这个站点和内容方向。"
comments: false
---
```

| 字段 | 要求 |
| --- | --- |
| `title` | 页面标题，必填。 |
| `description` | 页面摘要，用于 SEO。 |
| `comments` | 是否开启评论，默认 `false`。当前只保留字段，不开发评论 UI。 |

页面不要写 `slug`、`date`、`categories`、`tags`、`cover`、`top`。这些字段只属于文章。

## 正文图片引用

页面专用图片放在当前页面目录的 `img/` 内，用相对路径引用。

```md
![图片说明](./img/about-1.webp)

![图片说明](./img/about-2.webp)
```

如果图片会被多个页面或文章复用，可以放在 `blog/images/`。该目录构建后使用根路径访问，例如 `blog/images/banner.webp` 在 Markdown 中写作 `/banner.webp`。

## 提交规则

用户新增页面默认不提交到 Git。仓库中只保留 `blog/pages/about/**` 作为示例页面。
