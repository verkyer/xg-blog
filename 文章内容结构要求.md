# 文章内容结构要求

本文件用于迁移文章内容。只描述文章目录、Markdown frontmatter 和图片命名规则。

## 目录结构

每篇文章一个独立目录，目录名使用文章 slug。封面和正文图片统一放在 `img/` 目录内。

```text
blog/posts/
└─ {slug}/
   ├─ index.md
   └─ img/
      ├─ cover.*        # 可选专用封面，优先级：svg > avif > webp > png > jpg > jpeg
      ├─ {slug}-1.*     # 没有专用封面时，默认作为封面 fallback
      ├─ {slug}-2.*
      └─ {slug}-3.*
```

## 文件命名

| 文件 | 要求 |
| --- | --- |
| `index.md` | 文章正文和 frontmatter。 |
| `img/cover.*` | 可选专用封面图；优先级：`svg > avif > webp > png > jpg > jpeg`，新文章建议优先用 `.webp`。 |
| `img/{slug}-1.*` | 正文第 1 张图片；没有专用封面时，也作为封面 fallback。 |
| `img/{slug}-2.*` | 正文第 2 张图片。 |
| `img/{slug}-3.*` | 正文第 3 张图片，后续按顺序递增。 |

建议图片统一使用 `.webp`。如果迁移成本较高，也可以先保留原格式；示例文章使用 `.svg` 方便直接提交。

## Frontmatter

有专用封面时：

```md
---
title: "文章标题"
slug: "post-slug"
description: "文章摘要。"
date: "2026-05-29"
categories:
  - ai
tags:
  - astro
  - markdown
cover: "./img/cover.webp"
top: 0
comments: false
---
```

没有专用封面时，可以不写 `cover`，程序会自动读取 `img/{slug}-1.*`。

| 字段 | 要求 |
| --- | --- |
| `title` | 文章标题，必填。 |
| `slug` | URL 标识，建议只使用小写字母、数字和短横线。 |
| `description` | 文章摘要，用于列表和 SEO。 |
| `date` | 发布日期，使用 `YYYY-MM-DD`。 |
| `categories` | 分类数组，至少一个；建议使用小写 slug。 |
| `tags` | 标签数组，可为空；建议使用小写 slug。 |
| `cover` | 可选。建议写 `./img/cover.webp`；如果使用第一张正文图作为封面，则不写。 |
| `top` | 置顶权重，普通文章写 `0`。 |
| `comments` | 是否开启评论，默认 `false`。 |

## 正文图片引用

正文图片放在文章目录的 `img/` 内，按出现顺序命名。

```md
![图片说明](./img/post-slug-1.webp)

![图片说明](./img/post-slug-2.webp)
```

迁移时把 `post-slug` 替换成当前文章的真实 slug。

## 封面读取顺序

1. frontmatter 中的 `cover`。
2. `img/cover.*`（优先级：`svg > avif > webp > png > jpg > jpeg`）。
3. 兼容旧结构的根目录 `cover.*`（优先级：`svg > avif > webp > png > jpg > jpeg`）。
4. `img/{slug}-1.*`（优先级：`svg > avif > webp > png > jpg > jpeg`）。
5. `/default-cover.svg`。

默认封面文件位于 `blog/images/default-cover.svg`，构建后使用根路径 `/default-cover.svg` 访问。

## 分类和标签

分类和标签 slug 后续以小写为准。构建时会从文章 frontmatter 自动收集分类和标签；如需修改显示名称或描述，同步整理 `blog/categories.toml` 和 `blog/tags.toml`。
