# 文章内容结构要求

本文件用于整理用户文章内容。正式文章写在 `blog/posts/`；`example/posts/` 只放模板和空内容时的兜底示例，用户不要直接在 `example/` 里维护正式内容。

## 目录结构

每篇文章一个独立目录，目录名默认作为文章 URL slug。封面和正文图片统一放在当前文章目录的 `img/` 里。

```text
blog/posts/
└─ {slug}/
   ├─ index.md
   └─ img/
      ├─ cover.*        # 可选专用封面
      ├─ {slug}-1.*     # 正文第 1 张图片，也可作为备用封面
      ├─ {slug}-2.*
      └─ {slug}-3.*
```

建议图片优先使用 `.webp`。示例文章也使用 `.webp`。

## 文件命名

| 文件 | 要求 |
| --- | --- |
| `index.md` | 文章正文和属性区。 |
| `img/cover.*` | 可选专用封面；优先级：`svg > avif > webp > png > jpg > jpeg`。 |
| `img/{slug}-1.*` | 正文第 1 张图片；没有专用封面时，也作为备用封面。 |
| `img/{slug}-2.*` | 正文第 2 张图片。 |
| `img/{slug}-3.*` | 正文第 3 张图片，后续按顺序递增。 |

## 属性区

Markdown 文件开头两条 `---` 之间是属性区，用来写标题、日期、分类、封面等信息。属性区后面就是文章正文。

最小完整示例：

```md
---
title: "文章标题"
slug: "post-slug" # 当值为空或没有这个属性，默认使用文章文件夹名称
description: "文章摘要。"
date: "2026-06-09"
categories:
  - website
tags:
  - astro
  - markdown
cover: "./img/cover.webp"
top: 0
comments: false
---

这里写文章正文第一段，说明这篇文章要记录的内容。

## 小标题

正文可以继续写段落、列表、引用和代码块。`##`、`###`、`####` 会进入文章目录。

![文章配图](./img/post-slug-1.webp)
```

没有专用封面时，可以不写 `cover`，程序会继续尝试读取 `img/{slug}-1.*`。

| 字段 | 要求 |
| --- | --- |
| `title` | 文章标题，必填。 |
| `slug` | URL 标识，可选。空值或没有这个属性时，默认使用文章文件夹名称；建议只使用小写字母、数字和短横线。 |
| `description` | 文章摘要，用于列表和 SEO。 |
| `date` | 发布日期，使用 `YYYY-MM-DD`。 |
| `categories` | 分类数组，至少一个；建议使用小写 slug。 |
| `tags` | 标签数组，可为空；建议使用小写 slug。 |
| `cover` | 可选。建议写 `./img/cover.webp`；如果使用第一张正文图作为封面，则不写。 |
| `top` | 置顶权重，普通文章写 `0`。 |
| `comments` | 是否开启评论，默认 `false`。 |

## 正文图片

正文图片放在当前文章目录的 `img/` 中，用相对路径引用：

```md
![图片说明](./img/post-slug-1.webp)

![图片说明](./img/post-slug-2.webp)
```

迁移时把 `post-slug` 替换成当前文章的真实 slug。

## 正文链接

正文里的站内链接使用无域名的根路径，外部链接保留完整 URL。

| 链接类型 | 推荐写法 | 说明 |
| --- | --- | --- |
| 同一篇文章内锚点 | `#小标题` | 保持在当前文章页面内跳转，和文章目录组件一致。 |
| 其他站内文章 | `/archives/post-slug` | 去掉站点域名，不写尾部斜杠。 |
| 其他站内文章锚点 | `/archives/post-slug#小标题` | 仍使用根路径，锚点接在目标文章路径后。 |
| 外部链接 | `https://example.com/path` | 保留完整 URL。 |

不要把文章内同页锚点写成 `/#小标题`。这种写法指向站点根路径的锚点，在文章页会跳出当前文章。

## 封面读取顺序

1. 属性区中的 `cover`。
2. `img/cover.*`。
3. 文章根目录的 `cover.*`。
4. `img/{slug}-1.*`。
5. `/default/default-cover.webp`。

缺省封面文件位于 `defaults/public/default/default-cover.webp`，构建后使用 `/default/default-cover.webp` 访问。

## 分类和标签

分类和标签 slug 后续以小写为准。构建时会从文章属性区自动收集分类和标签；如需修改显示名称或描述，先从 `example/categories.toml`、`example/tags.toml` 复制到 `blog/` 根目录，再维护复制出来的 TOML 文件。

迁移文章时，分类和标签配置视为已经维护完成。迁移工具只需要把 Halo 文章里的分类和标签 slug 写入 frontmatter，不要为了迁移单篇文章额外检查或修改 `blog/categories.toml`、`blog/tags.toml`。
