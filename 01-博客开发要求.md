# 博客开发要求

本文档由 `xiaoge-static-blog-plan.md` 拆分整理，用于约束小鸽志静态博客主体开发范围。本文档只覆盖公开博客站点本身，不包含本地编辑器实现，也不包含评论、阅读数、点赞数等动态互动功能。

## 1. 项目目标

将现有动态博客改造为基于 Astro 的纯静态博客，在不使用数据库的前提下，尽量保持现有 URL、页面结构、SEO 信息和内容组织方式不变。

核心目标：

- 使用 Astro 作为静态博客底座。
- 使用 Markdown / MDX 管理文章内容。
- 使用 YAML 管理分类、标签、导航、友链等结构化配置。
- 图片优先跟随文章同目录管理。
- 源码、文章、图片和配置保存到 GitHub 私有仓库。
- 通过 Cloudflare Worker 或 GitHub Actions 自动构建部署。
- 保留旧站核心 URL，避免 SEO 损失和旧链接 404。

## 2. 技术选型

### 2.1 静态站框架

必须使用：

```text
Astro
```

选型原因：

- 支持 Markdown / MDX。
- 支持静态生成和自定义路由。
- 适合保留 `/archives/*`、`/categories/*`、`/tags/*` 这类旧路由。
- 适合技术博客、工具站、内容站。
- 后续可以按需添加前端交互组件。

### 2.2 数据存储

不使用数据库。所有内容必须以文件形式保存：

```text
Markdown 文章
YAML 分类配置
YAML 标签配置
YAML 导航配置
YAML 友链配置
图片文件
静态资源文件
```

## 4. URL 保持要求

URL 保持是本项目最重要的兼容要求。

### 4.1 文章 URL

旧站文章 URL：

```text
/archives/{slug}
```

新站必须继续使用：

```text
/archives/racknerd-2026-04
/archives/vmiss-2026-newyear
/archives/debian13
```

不得改为：

```text
/posts/{slug}
/blog/{slug}
/{year}/{month}/{slug}
```

实现位置：

```text
src/pages/archives/[slug].astro
```

实现逻辑：

```text
读取所有 Markdown
    ↓
提取 frontmatter.slug
    ↓
生成 /archives/{slug}
```

### 4.2 分类 URL

旧站分类 URL：

```text
/categories/{categorySlug}
```

新站必须继续保持：

```text
/categories/vps
/categories/vps-discount
/categories/vps-review
/categories/vps-tutorial
/categories/docker
/categories/linux
/categories/ai
```

### 4.3 标签 URL

旧站标签 URL：

```text
/tags/{tagSlug}
```

新站必须继续保持：

```text
/tags/cloudflare
/tags/racknerd
/tags/docker
/tags/linux
```

标签 URL 中的 slug 必须统一小写，显示名称可保留规范大小写。

### 4.4 集合页 URL

以下集合页必须保留：

```text
/archives
/categories
/tags
```

## 5. 文章内容模型

### 5.1 文章目录规范

必须采用“一篇文章一个文件夹”的组织方式：

```text
src/content/posts/racknerd-2026-04/
├── index.md
└── img/
    ├── cover.webp
    └── plans.webp
```

要求：

- 文章正文保存在 `index.md`。
- 文章相关图片保存在同级 `img/` 目录。
- 封面图统一优先命名为 `img/cover.webp`。
- Markdown 正文中使用相对路径引用文章图片。

示例：

```md
![RackNerd 套餐](./img/plans.webp)
```

### 5.2 Frontmatter 字段

每篇文章必须使用统一 frontmatter：

```md
---
title: "RackNerd 2026 四月 VPS 套餐更新：老套餐下架、新套餐涨价！"
slug: "racknerd-2026-04"
description: "RackNerd 2026 年四月 VPS 套餐更新，老套餐下架，新套餐价格上涨。"
date: "2026-04-25"
categories:
  - vps-discount
tags:
  - racknerd
  - us-vps
  - cheap-vps
cover: "./img/cover.webp"
top: 1
comments: true
---
```

字段说明：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `title` | 是 | 页面显示标题 |
| `slug` | 是 | 文章 URL 别名，用于生成 `/archives/{slug}` |
| `description` | 是 | SEO 描述和列表摘要 |
| `date` | 是 | 发布时间，支持 `20260425`、`2026-04-25`、`2026.04.25` 输入并统一渲染 |
| `categories` | 是 | 文章分类 slug 数组，允许多个 |
| `tags` | 否 | 文章标签 slug 数组 |
| `cover` | 否 | 封面图路径 |
| `top` | 否 | 置顶权重，数字越大越靠前，空值表示不置顶 |
| `comments` | 否 | 是否预留评论显示开关；首期不开发评论功能 |

### 5.3 分类规则

文章中只保存分类 slug：

```yaml
categories:
  - vps-discount
```

显示中文名称时读取：

```text
src/content/data/categories.yml
```

分类配置示例：

```yaml
- slug: vps-discount
  name: VPS优惠
  description: VPS 优惠活动、促销套餐、限时折扣
```

多分类规则：

- 一篇文章可以属于多个分类。
- 第一个分类作为主分类，显示在首页和文章卡片。
- 其他分类作为筛选分类，文章也必须出现在对应分类页。

### 5.4 标签规则

文章中只保存标签 slug：

```yaml
tags:
  - racknerd
  - us-vps
  - cheap-vps
```

显示规范名称时读取：

```text
src/content/data/tags.yml
```

标签配置示例：

```yaml
- slug: cloudflare
  name: Cloudflare

- slug: bandwagonhost
  name: 搬瓦工
```

## 6. 页面要求

### 6.1 首页

首页路径：

```text
/
```

首页必须包含：

- 最新文章列表。
- 置顶文章排序逻辑。
- 文章卡片标题、摘要、发布日期、主分类、标签、封面图。
- 侧边栏或等效区域，承载分类、标签、热门文章等原站常用信息。

排序规则：

```text
top 权重倒序
    ↓
date 倒序
```

### 6.2 文章详情页

文章路径：

```text
/archives/{slug}
```

文章页必须包含：

- 标题。
- 发布时间。
- 分类。
- 标签。
- 封面图。
- 正文内容。
- 文章目录 TOC。
- SEO meta 信息。
- canonical。

首期不要求实现：

- 动态评论提交。
- 动态阅读数。
- 动态点赞数。
- 实时评论数量。

### 6.3 归档页

归档路径：

```text
/archives
```

实现逻辑：

```text
读取所有文章
    ↓
按 date 倒序排序
    ↓
按年月分组
    ↓
生成归档列表
```

展示示例：

```text
2026年05月
  2026-05-06 腾讯云 LightVela 免费提供1个月的云端 Hermes Agent 服务
  2026-05-05 搬瓦工 纽约 CN2GIA 优化线路
```

### 6.4 分类集合页

路径：

```text
/categories
```

实现逻辑：

```text
读取 categories.yml
    ↓
统计每个分类下的文章数量
    ↓
生成分类集合页
```

每个分类项必须包含：

- 分类名称。
- 分类描述。
- 文章数量。
- 分类页链接。

### 6.5 分类详情页

路径：

```text
/categories/{categorySlug}
```

必须展示该分类下所有文章，按置顶权重和发布时间排序。

### 6.6 标签集合页

路径：

```text
/tags
```

实现逻辑：

```text
读取所有文章
    ↓
统计所有标签出现次数
    ↓
结合 tags.yml 显示规范名称
    ↓
生成标签列表或标签云
```

### 6.7 标签详情页

路径：

```text
/tags/{tagSlug}
```

必须展示该标签下所有文章，按置顶权重和发布时间排序。

### 6.8 独立页面

至少预留：

```text
/about
/links
```

## 7. 图片处理要求

### 7.1 新文章图片

新文章图片必须优先放入对应文章目录：

```text
posts/{slug}/img/
```

推荐格式：

```text
截图：webp
封面：webp
Logo：svg
动图：gif 或 mp4
透明图：png 或 webp
```

### 7.2 旧文章图片

迁移第一阶段不得强行大批量改写旧图片路径。应先完整保留旧图片访问路径，避免旧文章图片 404。

如果旧图路径是：

```text
/upload/2025/08/debian13.jpg
```

新站应尽量保留：

```text
site/public/upload/2025/08/debian13.jpg
```

后续可逐步整理为：

```text
posts/debian13/img/debian13.jpg
```

## 8. SEO 要求

每篇文章必须尽量保留或生成：

```text
title
description
canonical
datePublished
dateModified
og:title
og:description
og:image
twitter:title
twitter:description
sitemap
rss
```

SEO 验收重点：

- 旧文章 URL 不变。
- 旧分类 URL 不变。
- 旧标签 URL 不变。
- 旧图片不 404。
- `sitemap.xml` 可生成并包含文章 URL。
- `rss.xml` 可生成并包含最新文章。
- canonical 指向当前文章正式 URL。
- meta description 优先使用 frontmatter.description。

## 9. 部署要求

推荐部署链路：

```text
GitHub 私有仓库
    ↓
Cloudflare Worker / GitHub Actions
    ↓
www.xiaoge.org
```

Cloudflare 构建配置：

```text
Framework preset: Astro
Build command: npm run build
Build output directory: dist
```

构建产物必须包含：

```text
HTML
CSS
JS
图片
sitemap.xml
rss.xml
robots.txt
```

## 10. 迁移阶段要求

### 第一阶段：数据盘点

- 导出现有所有文章。
- 导出现有分类。
- 导出现有标签。
- 导出现有图片。
- 导出现有页面。
- 整理旧链接清单。
- 整理旧图片路径清单。

### 第二阶段：搭建 Astro 基础结构

- 创建 Astro 项目。
- 建立文章 Content Collection。
- 建立 `/archives/[slug]`。
- 建立 `/categories/[slug]`。
- 建立 `/tags/[slug]`。
- 建立归档页、分类集合页、标签集合页。

### 第三阶段：导入文章

- 转换文章为 Markdown。
- 保留原 slug。
- 保留发布时间。
- 保留分类。
- 保留标签。
- 处理图片路径。

### 第四阶段：还原主题样式

- 还原首页。
- 还原文章列表卡片。
- 还原文章页布局。
- 还原侧边栏。
- 还原热门文章。
- 还原文章目录。
- 还原底部信息。

### 第五阶段：处理 SEO

- 生成 sitemap。
- 生成 RSS。
- 检查 canonical。
- 检查 meta description。
- 检查 Open Graph。
- 检查旧链接访问状态。

### 第六阶段：部署测试

- 部署到测试域名。
- 抓取旧链接对比。
- 检查 404。
- 检查图片。
- 检查移动端。
- 检查页面速度。
- 检查搜索引擎索引风险。

### 第七阶段：正式切换

- 备份旧站。
- 冻结旧站文章发布。
- 最后一次导出文章。
- 构建静态站。
- 切换域名。
- 观察日志和 404。
- 补充重定向规则。

## 11. 非目标

首期博客主体开发不包含：

- 本地 FastAPI 编辑器。
- 评论系统。
- 阅读数统计。
- 点赞系统。
- 动态评论数量统计。
- 用户登录系统。
- 管理后台在线发布。
- 数据库存储。

## 12. 验收标准

开发完成后至少满足：

- `npm run build` 可以成功生成静态站。
- `/archives/{slug}` 可访问迁移后的文章。
- `/archives`、`/categories`、`/tags` 可访问。
- 分类详情页和标签详情页可访问。
- Markdown frontmatter 解析正确。
- 分类、标签中文名读取 YAML 配置显示。
- 文章封面和正文图片正常显示。
- 旧图片路径不 404。
- sitemap、RSS、robots 文件可生成或正常提供。
- 旧 URL 清单抽样访问无 404。
